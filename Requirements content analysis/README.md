# 需求内容分析

> **Skill 名称**：`requirements-content-analysis`
> **适用角色**：交互设计师、UX 设计师、产品设计师
> **核心价值**：将原始 PRD 快速转化为结构清晰、设计导向明确的需求分析报告

---

## 这个 Skill 是什么

`requirements-content-analysis` 帮助设计师在拿到 PRD 或需求描述后，快速完成结构化分析，输出一份包含以下八个模块的需求分析报告：

| # | 模块 | 核心问题 |
|---|------|----------|
| 一 | 需求背景 | 为什么要做这件事？ |
| 二 | 目标用户与用户场景 | 我们为谁设计？在什么场景下？ |
| 三 | 设计目标 | 要达到什么效果？用户关键路径是什么？ |
| 四 | 竞品分析 | 同类产品是怎么做的？差距在哪？ |
| 五 | 设计策略 | 综合分析后，具体怎么做？ |
| 六 | 设计原则与方向 | 项目的设计准则是什么？ |
| 七 | 设计方案交付 | 交付物清单是什么？ |
| 八 | 设计验证 | 如何衡量这次设计是否成功？ |

## 快速上手

### 使用方式

将你的需求描述或 PRD 内容直接发给 Claude，并说明你想生成需求分析报告，Skill 会自动引导输出对应结构。

**触发示例：**
- "帮我分析这个需求，生成设计分析报告"
- "根据这份 PRD，帮我整理需求分析"
- "我有一个新功能需求，帮我做需求拆解"

### 适用场景 ✅

- 产品新功能 / 新模块的设计需求分析
- 现有功能体验优化的前期分析
- 快速整理 PRD，梳理设计方向

### 不适用场景 ❌

- 微小 UI 调整（直接改稿更高效）
- Bug 修复
- 已有明确方案的小迭代

## 参考资料说明

Skill 在分析过程中会按需引用 `references/` 目录中的三份知识文档：

### `Cognitive Psychology.md` — 认知心理学
包含希克定律、菲茨定律、米勒定律、格式塔原理、峰终定律等核心认知原则。
**适用于**：分析用户决策成本、信息分组策略、交互复杂度判断。

### `Interaction-design-principles.md` — 交互设计原则
包含 Nielsen 十大可用性启发式、格式塔视觉原理、通用交互定律及设计评审检查清单。
**适用于**：评审设计方案、识别可用性风险、为设计决策提供原则依据。

### `Multi-Platform Design Principles.md` — 多端设计差异
涵盖移动端与 PC 端在布局、输入方式、导航、手势、性能、视觉等维度的差异对照。
**适用于**：当需求涉及多个平台端时，辅助判断各端的设计策略差异。

> 💡 当用户的需求明确指定了产品平台（如「移动端 App」「PC 后台」），Skill 会自动参考多端设计原则进行差异化分析。

## 输出格式说明

- **报告长度**：建议控制在 2–3 页以内，聚焦重点
- **内容深度**：根据需求信息的完整程度自动调整，信息充足时输出完整八节，信息不足时会主动询问补充
- **视觉建议**：报告中如有流程图、用户路径表格、竞品对比表，会以 Markdown 表格形式呈现

## 最佳实践

**分析前准备：**
- 提供业务背景和触发原因（为什么现在做）
- 明确目标用户群体
- 如果有竞品参考，一并提供

**常见问题规避：**
- 不要只描述「想要的功能」，优先描述「要解决的用户问题」
- 明确技术或时间限制，避免分析脱离实际约束
- 有历史数据或用户反馈时，一并提供以提升分析质量
