# 交互说明案例（Complete Examples）

本文档提供完整的交互说明写作案例，供撰写时参照格式使用。

---

## 一、组件交互说明模板

### 模板A：按钮

```markdown
**[按钮名称]**
- 默认态：[颜色/文字]
- 悬停态（Web）：[变化描述]
- 按压态：[变化描述]
- Loading 态：显示 loading 图标，禁止点击，防止重复提交
- 禁用态：[触发条件] + 置灰不可点击
- 点击行为：[触发条件] → [系统响应] → [反馈方式]
```

### 模板B：表单输入项

```markdown
**[字段名]**
- 类型：单行文本 / 多行文本 / 数字 / 选择器
- 占位提示：「请输入xxx」
- 最大长度：XX 字符（超出时截断输入 / 不允许继续输入）
- 校验规则：[实时校验 / 失焦校验 / 提交时校验]
- 错误提示：显示在输入框下方，文字「xxx」，输入框变红色边框
- 清空按钮：有内容时右侧显示 ✕，点击清空内容并聚焦
```

### 模板C：列表

```markdown
**[列表名称]**
- 加载中：展示骨架屏，共 N 行占位
- 有数据：[每行布局描述]，支持[上拉加载更多 / 分页]
- 空状态：展示[插画 + 文案「暂无xxx」]，[是否展示引导按钮]
- 下拉刷新：[支持 / 不支持]，刷新动画[描述]
- 列表项点击：[跳转 / 展开 / 弹窗]
- 列表项长按：[无反应 / 出现操作菜单]
```

### 模板D：弹窗 / 抽屉

```markdown
**[弹窗名称]**
- 触发方式：[操作描述]
- 出现动效：[从中间缩放出现 / 从底部滑入]，时长 [N]ms
- 遮罩：[有 / 无]，点击遮罩[关闭 / 不关闭]
- 关闭方式：[点击关闭按钮 / 点击遮罩 / 滑动关闭 / 仅按钮操作]
- 关闭动效：与出现动效方向相反，时长 [N]ms
- 内容滚动：弹窗内容超出时[内部滚动 / 撑开弹窗高度至最大值 N]
```

## 二、完整页面交互说明案例

### 案例：用户登录页面

```markdown
# 用户登录页面交互说明

**手机号输入框**
- 必填选项
- 输入规则：
  - 仅允许输入数字，限制最大 11 位
  - 超出 11 位时截断输入，不允许继续输入
  - 占位提示文字：「请输入手机号」
- 校验规则：输入框失焦校验，是否符合手机号格式
  - 不符合：toast提示"请输入正确手机号"

**获取验证码按钮**
- 显示规则：手机号格式正确时高亮显示
- 点击操作：文字变为「发送中...」，按钮禁用
  - 发送成功：Toast「验证码已发送」；按钮进入 60s 倒计时
  - 发送失败：Toast 显示具体失败原因（如「今日获取次数已上限」）

**验证码输入框**
- 必填选项
- 输入规则：
  - 仅允许输入数字，限制最大 6 位
  - 占位提示文字：「请输入验证码」
- 校验规则：登录按钮点击时，是否已填写且为 6 位数字
  - 不符合：toast提示"请输入正确的验证码"

**登录按钮**
- 显示规则：默认置灰不可点击；全部符合后高亮显示
- 点击操作：进行校验内容
  - 成功：Toast「登录成功」，跳转至首页
  - 失败：显示具体错误信息（如「验证码错误或已过期」）
- 校验规则：
  - 手机号格式
  - 验证码已填写
```
