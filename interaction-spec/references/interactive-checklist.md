# 交互说明自查规范

> 本文档用于：
> 1. **撰写交互说明时的结构参照**——确保说明文档覆盖所有必要维度
> 2. **交互说明完善性评审**——逐条检查是否存在遗漏，可直接作为走查 checklist 使用

## 一、目标分析

> 在开始任何交互设计前，需先确认以下目标是否有明确定义。

| 检查维度 | 检查点 | 说明 | 状态 |
|---|---|---|---|
| 业务目标 | 想要实现怎样的目标和成果？驱动动机是什么？ | 根据具体项目内容设定，但一定要搞清楚 | |
| 用户目标 | 针对什么用户？在什么场景？解决什么问题？达到什么目的？ | | |
| 产品目标 | 能否用一句可衡量的话描述？ | | |
| 设计目标 | 是否提炼了 1~3 个关键词并进行了解读分析？ | 根据以上三个目标提炼 | |

## 二、整体结构与信息架构

| 检查维度 | 检查点 | 说明 | 状态 |

| 信息分类 | 信息组织是否具有逻辑性？不同类型的信息区分是否清晰？ | | |

| 重点突出 | 最需要用户关注的内容是否放在最显眼的位置？ | | |
| 可扩展性 | 后续框架与内容扩展时，当前设计能否支撑？ | | |

## 附：快速自查 TOP 15（高频遗漏项）

> 以下是交互说明中最容易忽略的问题，优先逐一排查：

1. **空状态** — 列表、搜索结果、个人中心等是否都有空态设计？
2. **加载失败** — 每个接口请求失败时的兜底方案是否覆盖？
3. **权限差异** — 不同角色看到的页面和功能差异是否全部说明？
4. **控件展开后的操作** — 弹窗、抽屉、下拉菜单点开后的交互是否完整？
5. **流程中断与恢复** — 任务过程中被打断，恢复后从哪里继续？
6. **逆向流程** — 返回、取消、关闭后的状态和跳转是否符合预期？
7. **边界值** — 最长文本、最大数量、最小金额等极限情况下的展示？
8. **操作容错** — 用户误操作或放弃时，数据是否被保留？
9. **按钮禁用态** — 何时禁用、何时恢复？禁用时是否有 Tooltip 说明原因？
10. **网络异常** — 断网、弱网、超时的不同处理方式是否分别说明？
11. **危险操作二次确认** — 删除、清空、不可逆操作是否有确认弹窗？
12. **跳转名称一致性** — 点击前后的页面名称是否一致？
13. **通知授权** — 需要推送通知时，授权拒绝后的降级方案？
14. **输入框键盘行为** — 回车、Tab、Esc 等键盘操作是否有对应处理？
15. **内容过期** — 过期内容的展示方式和提示文案是否定义？
