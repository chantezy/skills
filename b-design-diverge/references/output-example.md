# 输出示例（快速出图模式）

> 场景：设计一个「订阅组成员」选择组件

---

**用户输入**：我想设计一个订阅组成员的选择组件，成员按部门分组，可以多选

---

### 方案A · 复选框列表

```
◉ 订阅组成员
    ☑ 产品部订阅组
    ☑ 研发部订阅组
    □ 市场部订阅组

○ 所有成员
```

适合：选项 < 20
优势：状态一目了然，学习成本极低
劣势：选项多时列表过长，无搜索能力

---

### 方案B · 穿梭框

```
┌──────────────────┐        ┌──────────────────┐
│ 待选 (12)        │        │ 已选 (3)         │
│ [🔍 搜索______]  │  [>]   │                  │
├──────────────────┤  [<]   ├──────────────────┤
│ □ 产品部订阅组   │        │ ☑ 研发部订阅组   │
│ □ 市场部订阅组   │        │ ☑ 运营部订阅组   │
│ □ 财务部订阅组   │        │ ☑ 法务部订阅组   │
└──────────────────┘        └──────────────────┘
```

适合：选项 > 20，需要搜索
优势：已选项持续可见，支持批量操作
劣势：组件较重，移动端适配差

---

### 方案C · 树形多选

```
[请选择订阅组 ▾]

  ▼ 全公司
    ▼ 技术中心
      ├─ ☑ 产品部订阅组
      ├─ ☑ 研发部订阅组
      └─ □ 测试部订阅组
    ▶ 运营中心

已选：产品部、研发部  [×] [×]
```

适合：有层级结构
优势：层级关系清晰，父节点全选高效
劣势：层级过深时操作繁琐

---

### 对比

```
           方案A      方案B      方案C
适用量级   < 20       20~100     有层级
操作步骤   直接选     搜索+选    展开+选
推荐       ⭐⭐⭐⭐    ⭐⭐⭐     ⭐⭐⭐⭐
```

想深入哪个方向？

> 场景：设计一个「数据看板配置」页面

---

**用户输入**：我想设计一个数据看板的配置页面，用户可以拖拽添加图表组件，调整布局

---

### 方案A · 画布式拖拽

```
┌─────────────────────────────────────────┐
│ [保存] [预览] [撤销] [重做]     配置面板 │
├─────────────────────────────────────────┤
│ ┌──────┐ ┌──────────┐ ┌──────┐        │
│ │图表1 │ │  图表2   │ │图表3 │        │
│ └──────┘ └──────────┘ └──────┘        │
│ ┌──────────────────┐ ┌──────┐        │
│ │     图表4        │ │图表5 │        │
│ └──────────────────┘ └──────┘        │
└─────────────────────────────────────────┘
```

适合：高度自定义布局
优势：布局自由度最高，适合创意型看板
劣势：学习成本高，响应式适配复杂

---

### 方案B · 栅格系统

```
┌─────────────────────────────────────────┐
│ [添加组件] [布局设置] [数据源管理]       │
├──────┬──────┬──────┬──────┬──────────────┤
│ 1/4  │ 1/4  │ 1/4  │ 1/4  │    操作      │
├──────┼──────┼──────┼──────┼──────────────┤
│图表1 │图表2 │图表3 │图表4 │   [编辑]     │
├──────┴──────┼──────┴──────┼──────────────┤
│    1/2      │    1/2      │   [编辑]     │
│   图表5     │   图表6     │   [删除]     │
└─────────────┴─────────────┴──────────────┘
```

适合：规范布局，易于维护
优势：布局规范统一，响应式适配简单
劣势：灵活性受限，不适合复杂创意布局

---

### 方案C · 模板预设

```
┌─────────────────────────────────────────┐
│ [选择模板] [自定义] [保存为模板]         │
├─────────────────────────────────────────┤
│ 模板预览：                              │
│ ┌─────────────────────────────────────┐ │
│ │ ┌──────┐ ┌──────┐ ┌──────┐        │ │
│ │ │KPI   │ │趋势图│ │表格  │        │ │
│ │ └──────┘ └──────┘ └──────┘        │ │
│ │ ┌──────────────────┐ ┌──────┐      │ │
│ │ │     详情         │ │饼图  │      │ │
│ │ └──────────────────┘ └──────┘      │ │
│ └─────────────────────────────────────┘ │
│ [使用此模板]                            │
└─────────────────────────────────────────┘
```

适合：快速搭建标准看板
优势：上手最快，模板质量有保证
劣势：自定义程度低，不适合特殊需求

---

### 对比

```
           方案A      方案B      方案C
灵活性     高         中         低
开发成本   高         中         低
推荐       ⭐⭐⭐     ⭐⭐⭐⭐    ⭐⭐⭐⭐
```

想深入哪个方向？
