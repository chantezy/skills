## Trello

**官方资料来源**：https://trello.com/guide

### 产品定位
最经典的卡片式 Kanban，极简主义，适合个人和小团队快速上手。官方定义："视觉化工作管理工具，帮助团队在协作、高效、有序的方式下共同构思、规划、管理和庆祝工作。"

### 核心结构（4个基础组件）
官方帮助文档将 Trello 的基础结构明确定义为：

**A. Board（看板）**
- 代表一个地方来追踪信息——通常对应一个大型项目、团队或工作流
- 不同场景下的 Board 完全独立，没有项目层级的概念

**B. List（列表/泳道）**
- 让卡片（任务）在不同进度阶段保持组织状态
- 可创建代表工作流各步骤的列，卡片在列之间流动
- 列数量不限，标题和排列顺序完全自定义

**C. Card（卡片）**
- 最小单元，代表一个任务或想法
- 卡片可包含：标题、描述（支持 Markdown）、成员、彩色标签、截止日、附件、清单（Checklist）、评论
- 核心交互：拖拽卡片跨列移动 = 状态流转
- 批量创建：复制列表/表格内容粘贴到卡片，Trello 自动将每行转换为独立卡片

**D. Board Menu（看板菜单）**
- 右侧控制中心：管理成员权限、设置、搜索卡片、启用 Power-Ups、创建自动化
- 活动日志：所有看板操作历史记录

### Power-Ups（扩展机制）
Trello 核心保持极简，通过 Power-Up 插件扩展能力：
- 日历视图、时间轴（甘特）视图
- 其他工具集成（Slack/GitHub/Jira等）
- 规则：免费版 1 个 Power-Up，Standard/Premium 无限

### 自动化（Butler）
- 基于规则的自动化：当 [条件] 发生时，执行 [动作]
- 条件示例：卡片移入某列、成员被指派、截止日临近
- 动作示例：移动卡片、添加标签、发送通知、归档卡片
- 配置使用自然语言描述，无需代码

### 多视图（Premium 功能）
- 看板视图（默认）
- 时间轴视图（甘特式）
- 日历视图
- 表格视图
- 地图视图

### 权限体系
- Board 级：私有 / 工作区可见 / 公开
- 成员角色：管理员 / 普通成员
- 企业版支持更细粒度的组织权限控制

### 最新功能（2025年）
- **Inbox**：从邮件/Slack 等直接捕获信息到 Trello
- **Planner**：同步日历，为任务分配专注时间段

### 值得参考的设计决策
- **极简三层结构**：Board → List → Card，无需学习成本，上手即用
- **拖拽=流转**：操作与状态变化的映射关系清晰直觉
- **卡片弹层而非跳转页面**：打开卡片是弹窗，不丢失看板上下文，是 Kanban 产品卡片交互的标杆设计
- **Power-Up 扩展模型**：核心极简 + 插件扩展，避免核心产品功能臃肿；适合参考"能力分层"的产品架构思路
- **批量创建卡片**：粘贴多行文本自动分拆为多张卡片，是高效初始化看板的实用设计

## PingCode

**官方资料来源**：https://pingcode.com/solutions/kanban-manage / https://docs.pingcode.com/agile/

### 产品定位
面向研发团队的专业敏捷开发管理工具，支持 Scrum 和 Kanban，覆盖需求→开发→测试→发布全流程，强调研发效能数据度量。

### Kanban 产品定位（官方原文）
"PingCode Kanban 管理为产研、PMO、业务等团队，提供灵活透明、轻量高效的工作模式。通过可视化和拉动的方式，帮助识别、改进团队瓶颈和问题，加速协作效率和持续交付能力。"

### Kanban 核心结构
**项目 → 看板 → 栏（列）→ 工作项（卡片）**

- **多看板支持**：一个项目内可创建多个看板（多版本并行开发看板、业务需求看板、技术支持看板等），看板间快速切换
- **泳道（Swimlane）**：对工作进行分类（按成员/优先级/类型等维度），让团队清楚哪些工作优先处理

### 默认工作流
PingCode Kanban 提供开箱即用的默认流程：
**需求池 → 设计 → 研发 → 测试 → 发布**

每个栏可拆分为"进行中"和"已完成"子状态，确保 DoD（Definition of Done 完成的定义）被满足后才流转到下一栏。

### WIP 限制（在制品限制）——核心设计
这是 PingCode Kanban 的核心差异化能力之一：
- 在「看板设置-栏设置」中为每个栏设置最小/最大工作项数量
- WIP 超出限制时，**看板栏背景色高亮变红**，视觉预警
- 拉动机制：下游顺畅时再从上游拉入新工作，避免大量任务并行
- 设计原则：建议单个任务工作时间控制在 8h 以内，防止大型工作项造成瓶颈

### 工作项模型
- 支持自定义工作项类型（用户故事/任务/缺陷等）
- 工作项可拆分子工作项
- 本地配置：开启后，项目内工作项类型独立配置，不受全局配置影响
- 自定义字段：支持工作项属性自定义，适配不同业务场景

### 自动化（触发器）
在「看板设置-触发器」中创建自动化规则：
- 典型场景：子任务全部完成时自动更新父任务状态、自动创建分支、自动配置权限
- 提供自动化模板库，Kanban 项目常用规则开箱即用

### 研发效能度量
提供 10 余种可视化报表：
- **累积流图（CFD）**：跟踪各栏工作项的累积数量变化，识别积压瓶颈
- **WIP 周期报告**：分析工作项在各阶段的停留时间
- **成员负荷报告**：每个成员的任务分配情况
- **缺陷统计图**：质量问题反馈

### 研发集成
- 代码、持续集成、测试用例、缺陷、文档均可关联对应工作项
- 支持 GitHub/GitLab/Gitee 集成，Commit/PR 关联任务
- 与 PingCode 测试管理、知识管理模块数据互通

### 权限体系
- 组织 → 项目 → 模块级别
- 角色：管理员 / 开发 / 测试 / 产品 / 访客
- 支持自定义角色和权限

### 值得参考的设计决策
- **WIP 超限变红预警**：不是弹窗打断用户，而是背景色高亮——被动感知而非主动阻断，体验更流畅
- **默认工作流 + 本地配置**：提供最佳实践的默认流程，但允许项目单独定制，"开箱即用"与"灵活配置"的平衡做法
- **栏拆分 Doing/Done**：每个栏内部再分"进行中"/"已完成"，工作项满足 DoD 才进入"已完成"等待流转，避免"以为完成但实际未完成"的状态混乱
- **累积流图（CFD）**：时序上直观展示各栏积压情况，是看板效能分析的标配可视化工具，建议作为数据分析模块的参考

## Teambition

**官方资料来源**：https://support.teambition.com/help/docs/60c8648168465f00468fb2a9（动态页面，无法直接抓取）

### 产品定位（基于搜索整理）
阿里系项目协作工具，任务管理 + 文件共享 + 甘特图，偏向项目全生命周期管理，与钉钉生态深度集成。

### 核心结构
**项目 → 任务列表/看板 → 任务**

项目是完整协作空间，包含成员、任务、文件、日程等。

### 主要特性
- **看板 + 列表双模式**：支持随时切换，列表适合有顺序的任务，看板适合状态流转管理
- **甘特图**：内置时间依赖管理，任务有开始/截止日期，支持依赖关系连线
- **子任务**：支持多层嵌套，适合复杂任务拆解
- **文件管理**：项目内置文件空间，文件可与任务关联
- **钉钉集成**：任务变更推送钉钉消息

### 值得参考的设计决策
- **任务与文件联动**：文件空间内置于项目，任务附件和独立文件统一管理，避免附件散落
- **甘特与看板的切换**：两种视图共享同一套任务数据，切换无缝——是多视图设计的参考案例

## Monday.com

**官方资料来源**：https://support.monday.com/（国内用户常参考官网功能介绍）

### 产品定位
Work OS（工作操作系统），高度可定制的工作管理平台，适合各类团队和工作流类型，以自定义字段和多视图为核心差异化。

### 核心结构
**Workspace → Board（看板）→ Group（分组）→ Item（工作项）**

Board 本质是一张可自定义列的表格，每列是一个字段类型，极其灵活。

### 多视图（核心差异化）
同一 Board 数据可切换：
- 表格视图（默认）
- 看板视图
- 甘特视图
- 日历视图
- 地图视图
- **工作量视图（Workload）**：可视化每个成员的任务负载，防止过载

### 自定义字段类型
字段类型极其丰富，包括：状态（彩色标签完全自定义）、人员、日期、数字、下拉、文本、关联（跨 Board）、镜像列（Mirror Column）等。

**镜像列（Mirror Column）**：将关联 Board 的字段数据镜像显示在当前 Board，不需要重复录入数据。

### 自动化（Automations）
- 可视化条件→动作配置，使用自然语言描述规则
- 支持跨 Board 的自动化
- 典型：当状态变为"完成"时，通知负责人并移动到下一 Group

### 仪表盘（Dashboard）
- 跨多个 Board 汇总数据
- 展示进度、工作量、图表等
- 适合管理层视角的项目总览

### 集成（Integrations）
与 Slack、GitHub、Jira、Google Suite 等大量工具双向同步。

### 权限体系
- Workspace → Board 级别
- Board 权限：私有/共享/公开
- Enterprise 版支持**列级别权限**（隐藏特定列）

### 值得参考的设计决策
- **状态列颜色完全自定义**：彩色标签强视觉区分，且颜色由用户自定义，不依赖产品预设——视觉表达与用户语义对齐
- **工作量视图（Workload）**：工作分配是看板的核心场景，而"负载是否均衡"是管理者的核心关切；这个视图直接解决了这个问题
- **镜像列（Mirror Column）**：跨 Board 数据同步显示，解决了跨项目信息孤岛问题，是跨系统联动的优秀参考
- **列级别权限（Enterprise）**：字段级权限隔离，适合同一份数据面向不同角色展示不同字段的场景

## TAPD（腾讯敏捷产品研发平台）

**官方资料来源**：https://www.tapd.cn

### 产品定位
腾讯自主研发的敏捷研发协作平台，沉淀腾讯 10 余年研发管理经验，覆盖需求规划、迭代跟踪、缺陷管理、DevOps 集成等全流程。面向中大型研发团队，强调"敏捷协作，创造新的可能"。

### 核心结构
**项目 → 迭代 → 工作项（需求/任务/缺陷/Bug）**

- **工作项管理**：灵活定制工作项类型，支持需求、任务、缺陷、Bug 等多种类型
- **迭代管理**：支持 Scrum 和 Kanban 双模式，覆盖迭代规划、执行、回顾全流程
- **需求池**：统一管理需求，支持优先级排序、评审流转

### 双流程引擎（核心差异化）
TAPD 提供两种工作流引擎，适配不同规模和复杂度的项目：

**A. 敏捷状态机模式**
- 适合小团队快速迭代
- 卡片在列之间拖拽流转
- 状态转换简单直观

**B. 多分支流程节点模式**
- 适合中大型团队复杂流程
- 支持条件分支、并行网关、会签等
- 流程可视化配置，支持自定义节点

### 自动化协作
基于「触发条件 + 执行动作」的规则引擎：
- **触发条件**：工作项创建/更新/删除、字段变更、时间触发等
- **执行动作**：更新字段、发送通知、创建关联工作项、调用 Webhook 等
- **规则模板**：提供常用自动化模板，开箱即用

### DevOps 集成
- 与主流研发工具深度集成：GitLab、GitHub、Jenkins、SonarQube 等
- 可视化交付流水线管理
- 代码提交关联工作项，自动触发构建/部署
- 支持接入企业自建工具

### 跨项目协作
- **联合项目**：多个子项目组成大型项目，统一管理
- **项目组合管理**：支持父子层级、跨组织协作
- **依赖关系**：工作项之间建立前后置依赖

### 权限体系
- 组织 → 项目 → 模块级别
- 角色：管理员 / 开发 / 测试 / 产品 / 访客
- 支持自定义角色和权限配置
- 按部门/项目组设置查看与编辑权限

### 企业微信集成
- 与企业微信深度打通，消息通知直达
- 支持企微机器人推送任务变更
- 移动端可通过企微小程序访问

### 安全能力
- ISO 27001 信息安全管理体系认证
- 网络安全等级保护
- 账号及认证安全、审计合规
- 异常警告、数据备份

### 值得参考的设计决策
- **双流程引擎**：敏捷状态机 + 多分支流程节点，覆盖从简单到复杂的全场景，"轻量"与"重量"模式自由切换
- **DevOps 全链路集成**：不只是任务管理，而是打通研发工具链，实现需求→代码→构建→部署的全链路可追溯
- **企业微信原生集成**：消息通知、移动办公、审批流程与企微无缝衔接，适合国内企业办公场景
- **项目组合管理**：支持大型项目的父子层级拆分和跨组织协作，适合中大型企业复杂项目管理需求

## 日事清

**官方资料来源**：https://www.rishiqing.com

### 产品定位
高效能团队管理专家，聚焦"任务管理 + 日程安排 + 工作汇报"三位一体，通过看板将工作任务合理分配至每位团队成员的个人日程，每天自动生成工作日志。适合中小团队提升执行力和工作透明度。

### 核心结构
**项目 → 看板 → 任务 → 子任务**

- **项目看板**：任务以卡片形式在看板上可视化展示
- **任务分配**：将看板任务指派给具体执行人，自动同步到个人日程
- **日程联动**：任务截止时间自动显示在个人日历中

### 核心特性

**A. 看板管理**
- 任务卡片支持拖拽移动
- 卡片可包含：标题、描述、执行人、截止日期、优先级、标签、附件
- 支持子任务分解，适合复杂任务拆解
- 卡片评论和讨论，文件云端无限存储

**B. 甘特图**
- 一键生成甘特图，无需专业技能
- 支持任务依赖关系（FS/FF/SS/SF）
- 实时更新，拖拽调整任务时间
- 适合项目进度管控和里程碑管理

**C. 日程管理**
- 个人日历视图，展示每日任务安排
- 任务与日程双向同步
- 支持按日/周/月查看工作计划

**D. 工作汇报**
- 每日自动生成工作日志（基于完成的任务）
- 支持手动补充工作内容和心得
- 汇报可分享给团队成员和上级
- 适合绩效考核和工作复盘

### WBS 分解
- 支持多层任务拆解，从项目到阶段到具体任务
- 父子任务关系清晰，进度自动汇总
- 适合复杂项目的结构化管理

### 风险管理
- 任务风险标记和预警
- 逾期任务高亮提醒
- 适合对进度敏感的项目场景

### 知识沉淀
- 任务附件和文档云端存储
- 支持团队共享知识库
- 项目经验可沉淀为模板复用

### 权限体系
- 项目级权限控制
- 角色：管理员 / 成员 / 访客
- 支持按项目设置查看/编辑权限

### 集成能力
- 支持微信/企业微信消息通知
- 支持钉钉集成
- 提供 API 接口，支持与其他系统对接

### 值得参考的设计决策
- **任务与日程联动**：看板任务自动同步到个人日历，解决了"任务管理"与"时间管理"割裂的问题，是"任务即日程"理念的优秀实践
- **自动工作汇报**：基于完成的任务自动生成日志，减少手动汇报的工作量，同时保证汇报内容的准确性
- **极简甘特图**：一键生成，无需复杂配置，降低甘特图使用门槛，适合中小团队快速上手
- **三位一体模型**：任务管理 + 日程安排 + 工作汇报形成闭环，从"分配任务"到"执行任务"再到"汇报成果"全流程覆盖
