## 飞书云文档

**官方资料来源**：https://www.feishu.cn/hc/zh-CN/category/7371372929647394819

### 产品定位
企业协作套件的内容中心，与飞书 IM、日历、会议深度集成，强调"全链条打通"。

### 核心文档类型
文档（富文本）、表格、思维笔记、幻灯片、多维表格、上传文件——统称"云文档"，权限体系统一管理。

### 多维表格（核心差异化能力）
飞书对多维表格的定位是"先进团队的业务管理工具"，而不只是表格，本质是轻量级低代码平台：
- **字段类型丰富**：数字、日期、单选、多选、附件、人员、关联等，可自定义编号规则
- **多视图**：数据表视图、看板视图、甘特视图、画册视图，同一份数据切换展示
- **仪表盘**：数据表级独立页面，集中展示图表，与数据表解耦
- **高级权限（行列级）**：配置角色组，细化到指定行/列/数据表的读写权限；列级别单独设置，适合跨部门协作时的数据隔离
- **自动化**：条件触发（时间/状态变更/新增记录）→ 执行动作（发消息/更新字段/创建记录等），打通飞书通知、日历、任务等
- **共享数据源**：可将某数据表发布为组织内共享数据源，其他多维表格可同步引用，选择字段和同步频率

### 云文档权限体系
- **协作者角色**：阅读者 / 评论者 / 编辑者 / 管理员（所有者）
- **分享控制**：可设置是否允许组织外分享、谁可以添加/移除协作者、是否允许复制/打印/下载
- **知识库权限**：知识库级别分管理员 / 可编辑成员 / 可阅读成员，三级角色
- **打通组织架构**：可按部门/用户组批量授权，新增员工自动继承对应权限

### 协同与通知
- 实时协同编辑，光标位置可见
- 评论支持 @ 成员，触发飞书消息通知
- 文档内可发起任务，任务状态与日历联动

### 值得参考的设计决策
- **共享数据源模式**：解决了传统共享文档"要么全开放要么数据不同步"的困境，一份数据，按需订阅
- **高级权限"千人千面"**：同一份表格，不同人看到不同行列数据，适合大规模多角色协作
- **自动化以业务事件为触发**：不依赖代码，运营/PM 可自行配置
- **多维表格仪表盘页面独立**：图表展示页和数据编辑页分离，避免视觉冲突

## 语雀

**官方资料来源**：https://www.yuque.com/yuque

### 产品定位
蚂蚁集团旗下知识创作与知识管理平台，核心理念是"像书一样的知识库"，强调内容结构化沉淀。

### 内容组织结构
- **空间**：最顶层容器，企业使用以团队空间为核心
- **知识库**：空间内的内容分组，类似"一本书"，有目录大纲
- **文档**：知识库内的页面，支持多级嵌套目录

文档类型：普通文档（富文本）、表格（在线表格）、演示文稿、绘图、数据表（类 Airtable）

### 编辑器能力
- 富文本 + Markdown 双模式
- 支持代码块、LaTeX 公式、PlantUML（流程图/时序图）
- 内链语法（`[[文档名]]`）实现文档间互相引用，形成双向知识网络
- 支持 Word/Excel/PPT 格式导入
- 版本控制：草稿态与发布态分离；历史版本可回溯对比

### 权限体系
- **知识库级**：公开（互联网可见）/ 私密（团队内）/ 加密（需密码访问）
- **成员权限**：管理员 / 普通成员（编辑）/ 只读
- **文档分享**：生成只读链接 或 设置为团队内可见，亦可发布到公开语雀社区
- 全文搜索：关键词搜索，结果按相关性排序

### 值得参考的设计决策
- **发布/草稿分离机制**：文档修改不立即影响读者，适合需要"审阅后再上线"的正式文档场景；语雀在这方面设计最成熟
- **目录大纲常驻侧边栏**：知识库结构始终可见，适合内容体量大的团队
- **内链知识网络**：双向引用形成知识关联，适合沉淀复杂知识体系；是其区别于普通文档工具的核心能力
- **加密知识库**：适合对外分发受控内容，比权限管理更轻量

## 石墨文档

**官方资料来源**：https://shimo.im/help

### 产品定位
国内最早专注实时协同编辑的文档工具，以协作体验和多端同步为核心，近年拓展了多种内容类型。

### 内容类型（帮助中心一级分类）
- 在线文档
- 表格
- **应用表格**（结构化数据管理，支持多种视图，类飞书多维表格）
- 表单（信息收集，结果汇入表格）
- 幻灯片
- 思维导图
- 白板

### 权限与协作
- **共享文件夹**：文件夹级别设置成员权限，区分编辑/查看
- **协作者 vs 链接分享**：两种模式明确区分，协作者适合持续协作，链接分享适合临时只读
- **企业版**：管理员角色、成员管理、席位管理
- 移动端支持历史版本回溯、评论查看

### 应用表格（差异化功能）
- 石墨"应用表格"是其近年主推的结构化数据功能，帮助文档独立分类管理
- 支持多种视图（表格/看板/表单等）、字段类型配置、与表单联动收集数据

### 值得参考的设计决策
- **权限模式的清晰区分**：协作者（持续协作）vs 链接分享（临时只读）这两条路径界面隔离清晰，用户无需理解复杂权限体系就能做出正确选择
- **内容类型边界清晰**：帮助文档为每种内容类型（文档/表格/应用表格/表单/幻灯片/思维导图/白板）独立维护，产品功能边界感强
- **思维导图内置**：不依赖第三方工具，适合写作前的思路梳理场景

## 有道云笔记

**官方资料来源**：https://note.youdao.com/help-center/

### 产品定位
网易旗下个人知识管理工具（2011年上线），强调个人内容沉淀、多平台同步和安全存储，近年向 AI 知识助手方向转型。

### 功能体系（完整帮助中心分类）

**基础功能**
- 笔记创建（普通笔记、脑图笔记）
- 内容收藏（网页/内容剪藏）
- 笔记分享
- 微信小程序支持

**进阶功能（近年密集迭代）**
- **AI 助手**：全功能 AI 对话工具，可基于笔记内容问答
- **AI 摘要**：一键生成文档摘要
- **文转图表**：文字描述自动生成图表（AI 驱动）
- **双链笔记**：笔记间互相引用，形成双向链接网络
- **白板笔记**：自由布局的白板，支持便签/图形/连线
- **语音速记**：语音转文字记录
- **每日回顾**：闪卡式内容复习（间隔复习理念）
- **收藏标注**：内容高亮和批注
- **待办功能**：笔记内嵌任务清单，与日历联动
- **PDF 转 Word**、**文字识别（OCR）**
- 笔记历史版本回溯

**拓展能力（开发者向）**
- OpenClaw（浏览器扩展剪藏）
- YoudaoNote CLI（命令行访问笔记）
- YoudaoNote Skills（基于知识库的 AI 技能，可自定义扩展）

### 存储与安全
- 初始免费 3G 存储，付费版扩容
- 三备份存储
- 客户端应用锁

### 值得参考的设计决策
- **每日回顾机制**：将间隔复习理念引入文档工具，强化知识留存，是"读写分离"场景的独特设计
- **AI 功能覆盖完整**：从 AI 摘要到文转图表到 AI 助手，是国内文档工具中 AI 功能较全面的
- **双链 + AI 的组合**：双链形成知识网络，AI 在此基础上做问答，构成知识管理的完整闭环
- **Skills 扩展机制**：允许用户在知识库上构建自定义 AI 技能，类似 Notion 的集成生态思路

## Notion

**官方资料来源**：https://www.notion.com/zh-cn/help

### 产品定位
All-in-one workspace，用 Block + Database 覆盖文档/Wiki/任务/数据库所有场景，近年大力布局 AI 能力和 Agent 生态。

### 核心架构
- **Block（块）是一切的基础**：文字、图片、代码、数据库均为 Block，可任意嵌套组合
- **Page（页面）**：一切内容的容器，Page 可无限嵌套，Database 本身也是 Page
- **Teamspace（团队空间）**：企业版协作单元，类似飞书知识库，有独立权限
- **Library**：侧边栏内容管理，统一视图管理所有页面

### Database 功能体系（Notion 核心差异化）
同一份数据库可以切换 11 种视图：
表格 / 看板 / 时间轴 / 日历 / 列表 / 画册 / 仪表盘 / 图表 / 地图 / 信息流 / **表单（Forms）**

表单视图可收集外部提交数据，直接写入数据库，无需独立表单工具。

**Database 高级特性**
- **Relations & Rollups**：不同数据库间可建立关联字段，并通过 Rollup 汇总关联数据（轻量级关系型数据库）
- **Linked Database（链接数据库）**：同一数据库可在多个页面以不同视图/筛选条件展示
- **Sub-items & Dependencies**：任务有子项，任务间支持依赖关系
- **Sprints（迭代）**：数据库支持 Sprint 功能，适合研发团队
- **Unique ID**：每条记录自动生成唯一编号
- **Database page-level access**：基于记录中的人员字段自动控制该记录的访问权限（2025年新功能）

### 协作与权限
- **Workspace → Teamspace → Page** 三级权限，逐层继承可单独覆盖
- **角色**：Full access / Can edit / Can comment / Can view / No access
- **Suggested edits（建议编辑）**：类似 Word 批注追踪，支持审核接受/拒绝
- **Page analytics**：查看页面访问数据

### 自动化（Automations）
- **模板按钮（Template Button）**：点击按钮批量执行操作（创建内容/修改字段等）
- **Database Automations**：条件触发 → 执行动作，支持 Webhook 外发
- **Filters/Sorts/Groups**：视图级筛选/排序/分组，不影响底层数据

### AI 功能（近年重点布局）
- **Notion AI**：文档内 AI 助手（写作/摘要/翻译/改写）
- **Notion Agent**：自动化工作的 AI 代理（2025年主推）
- **AI Meeting Notes**：会议记录 AI 整理
- **Enterprise Search**：跨工作区全局搜索
- **AI Connectors**：连接 GitHub / Google Drive / Slack / Jira / Microsoft Teams / Asana 等，实现 AI 跨平台内容检索
- **Custom Agents**：自定义 AI 代理，支持 MCP 协议连接外部工具

### 数据迁移与集成
- 支持从 Asana / Confluence / Monday.com 迁移数据
- Synced Databases（跨空间数据库同步）
- **Synced Block（同步块）**：同一段内容同步出现在多个页面，修改一处全部更新

### 值得参考的设计决策
- **Database 11 种视图架构**：一份数据，无限视角——这是 Notion 产品哲学的核心体现，也是多视图需求的最佳参考案例
- **Forms 视图内置于 Database**：收集 → 汇总 → 分析全链路在一个工具内完成，无缝体验
- **Database page-level access**：根据记录中的人员字段自动控制权限，适合任务/项目分配场景，权限随数据走
- **Synced Block**：同一内容块同步显示在多处，内容复用的极致设计
- **Agent + MCP 生态**：Notion 正在从文档工具向 AI 工作空间转型，Custom Agents 是其核心差异点
