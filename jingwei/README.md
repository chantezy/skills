# jingwei-product-discussion

> 精卫平台（4399 内部工具）需求设计方案讨论 Skill
> 适用于文档系统与项目管理系统的产品需求分析、方案发散、竞品参考。

---

## 这是什么

一个部署在 Claude 上的 Skill，让 AI 在讨论精卫平台相关产品需求时，能够：

- 结合平台现有能力给出**贴合实际**的方案，而不是通用产品建议
- 主动引用竞品的**成熟设计决策**作为参考（飞书/Notion/语雀/Trello/PingCode/TAPD/日事清/WPS笔记 等）
- 按照统一框架推进讨论：定性问题 → 还原场景 → 发散方案 → 给出推荐
- 适用于文档系统（精卫文档）和项目管理系统（精卫项目管理）两个方向

## 快速开始

### 使用方式

直接在对话中描述你遇到的产品问题，Skill 会自动触发。无需特殊指令。

**触发示例：**
```
用户反馈权限设置太复杂了，怎么优化？
我想做一个让看板卡片自动归档的功能
Notion 的数据库视图我们要不要做？
多人同时编辑文档会冲突怎么处理？
项目管理里能不能关联文档系统的内容？
```

## 文件说明

### SKILL.md
Skill 的核心文件，包含：
- 方案讨论的四步框架（问题定性 → 场景还原 → 方案发散 → 推荐）
- 文档系统和项目管理系统各自的方案输出规范
- 跨系统联动场景预设
- 竞品速查表
- 输出风格要求（有立场、贴平台、可继续讨论）

### platform-spec.md
精卫平台自身的产品规格说明，是方案讨论最重要的参考依据：
- 精卫文档的功能模块、页面结构、权限体系
- 精卫项目管理的功能模块、迭代生命周期、权限体系
- 两个系统的集成关系
- **当前已有能力** vs **暂未明确有的能力**（方案讨论的边界参考）

### competitors-docs.md
文档系统 6 个竞品的深度分析：
- 飞书云文档（多维表格、行列级权限、共享数据源）
- 语雀（发布/草稿分离、内链知识网络）
- 石墨文档（协同编辑体验、权限模式区分）
- 有道云笔记（AI 功能体系、每日回顾机制）
- Notion（Database 11 种视图、Synced Block、AI Agent）
- WPS笔记（AI原生多模态笔记、AI标签替代文件夹、从记录到复用全链路AI）

### competitors-pm.md
项目管理 6 个竞品的深度分析：
- Trello（极简三层结构、卡片弹层交互、Butler 自动化）
- PingCode（WIP 限制变红预警、累积流图、默认工作流）
- Teambition（任务与文件联动、甘特与看板切换）
- Monday.com（多视图、镜像列、工作量视图、列级权限）
- TAPD（双流程引擎、DevOps全链路集成、企微深度打通）
- 日事清（任务+日程+汇报三位一体、自动生成工作日志、极简甘特图）

## 更新维护

### 日常小更新（推荐）
直接在对话中告知 AI 最新情况：

```
我们文档系统新增了实时协同编辑，请以此为准
看板现在支持 WIP 限制了，platform-spec 里那条"暂未有"可以划掉
```

AI 会在当次对话中以你的描述为准，无需重新打包 Skill。

### 季度/大版本更新
1. 修改 `references/platform-spec.md` 对应章节
2. 重新打包文件
3. 在替换旧版本

竞品信息变化较慢，通常无需频繁更新；平台自身功能迭代较快，建议每季度更新一次 `platform-spec.md`。
