# interaction-design-eval

交互设计方案专业评估工具，用于评估不同职级（P2/P3/P4）设计师的方案价值与能力匹配度。

## 适用场景

- 上传设计方案文档、原型说明、PRD 交互部分，进行专业评审打分
- 设计师职级评估、成长值评估、能力匹配度盘点
- 团队内设计方案横向对比

## 评估体系

### 10 大评估维度

| 维度 | 权重 | 核心关注 |
|---|---|---|
| 业务目标匹配度 | 10% | 是否贴合核心业务指标、助力转化 |
| 用户理解能力 | 10% | 是否准确把握用户需求、动机和场景 |
| 创新性与价值表达 | 5% | 是否有创新思路、解决传统痛点 |
| 任务流程设计 | 15% | 是否帮助用户高效完成目标任务 |
| 可实现性评估 | 10% | 是否考虑技术落地可行性 |
| 交互逻辑完整性 | 15% | 是否覆盖各种场景和需求 |
| 信息架构与内容组织 | 10% | 是否帮助用户快速找到所需信息 |
| 交互细节表达 | 10% | 是否增强用户沉浸感和参与 |
| 可用性与可访问性 | 10% | 是否提供简单易用的操作方式 |
| 设计规范适配性 | 5% | 是否符合设计规范和标准 |

### 评分规则

- **分值**：每个维度 1~5 分，打 3 次取平均
- **成长值**：1~5 分，反映职级能力溢出与晋升潜力
- **合并分** = 分值 + 成长值
- **总分** = 各维度加权分之和 × 10，满分 100 分

### 职级标准

参照 P2/P3/P4 三个职级的 12 项能力标准（沟通能力、解决问题、执行力、项目管理、设计类、相关知识、设计定位、设计实施、设计总结、方法论建设、知识传承、人才培养）进行成长值评估。

## 输出内容

1. **评分表格**：10 维度的分值、成长值、合并分、权重、加权分
2. **问题清单**：每个维度 3 个以上突出问题，附原文引用
3. **优化建议**：针对问题的可执行改进方案
4. **成长值比例**：用于横向对比团队成员晋升潜力
