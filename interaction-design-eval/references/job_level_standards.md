# 交互设计岗位职级能力标准（P2 / P3 / P4）

## 使用说明

- 每个能力项目（如"沟通能力""设计定位"）在 P2/P3/P4 三个职级下有不同的关键词和行为标准
- 职级越高，要求的能力范围越广、自主性越强、影响力越大
- 评估时需先确认评估对象的职级（如 P3T1 中的"P3"），再对照该职级对应列的行为标准
- 如果评估对象的实际表现明显超出/低于其职级标准，应在评分说明中注明

## 一、通用能力

### 1. 沟通能力

| 职级  | 关键词          | 行为标准                                                                                                                     |
| --- | ------------ | ------------------------------------------------------------------------------------------------------------------------ |
| P2  | 表述自己的要点      | 1）有主动沟通的意愿，掌握基本沟通技巧，完成一般的目标单一、内容简单的沟通任务<br>2）能够清楚表达工作内容和个人观点<br>3）能够利用常用OFFICE软件、邮件、报表等进行日常工作汇报或交流                       |
| P3  | 把握他人论述要点     | 1）能准确无误、简练的表达自己的观点，把握他人的叙述要点<br>2）多数情况下都能够有效倾听和理解对方<br>3）能熟练应用PPT、邮件、报表等形式进行书面汇报，有逻辑地展示论据，论证观点，格式清晰规范<br>4）能够主持小型会议(5人内) |
| P4  | 多种沟通技巧，跨团队沟通 | 1）准确无误、逻辑清晰、简练的表达自己的观点，准确的领悟对方观点<br>2）掌握多种沟通技巧，能进行跨团队沟通，达成共同目标<br>3）能够主持中型会议(15人内)                                       |

### 2. 解决问题

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 解决例行及局部问题 | 1）独立处理和解决专业领域例行的问题<br>2）在工作中遇到的局部性问题上，能通过独立思考和推导，正确评估利弊做出决断 |
| P3 | 独立解决，通过经验评估方案 | 1）独立处理和解决专业问题<br>2）能有效收集信息和数据资料以解决在自己工作范围内所遇到的问题<br>3）能汲取过往经验有效评估不同解决方案所需要的资料和资源 |
| P4 | 理清因果，将复杂问题进行分解 | 1）确定问题的根源及背后的因果关系，并找出与问题和建议方案相关的风险所在，并采取适当的预防措施<br>2）将复杂的问题进行拆分，指出关键控制点，并系统性地从多个方案中选择最佳方案 |

### 3. 执行力

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 按要求行事 | 1）能遵循上级或计划对时间、步骤、方法、途径等的工作布置<br>2）按计划或指示的要求要点完成工作 |
| P3 | 领会意图，主动汇报 | 1）领会任务要到达的目的、意图、决策原因、适用情景等<br>2）当情况变化时，能及时向上级汇报<br>3）能够及时反馈与沟通任务进展情况 |
| P4 | 抓住重点，克服困难 | 1）能够承担有挑战性项目或工作任务<br>2）能够全面分析并抓住任务关键因素<br>3）克服困难，采取有效措施，高质量、高效率完成任务 |

### 4. 项目管理能力

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 指导下进行计划跟踪和监控 | 1）熟悉项目管理基础知识<br>2）能在指导下，对已制定好的项目计划进行跟踪和监控<br>3）在计划执行中参与一些辅助性的工作，协助解决问题 |
| P3 | 组织实施小型项目 | 1）组织实施小型项目，合理进行任务分解和进度安排<br>2）能够按照总体计划制定阶段性计划及监控点，并按实际执行情况及时修正项目计划<br>3）在项目进行中能及时发现并反馈问题，判断问题的重要程度并解决一般难度的问题 |
| P4 | 独立负责中型项目的实施和运作，预见潜在问题 | 1）能够独立负责中型项目的实施和运作，清楚了解项目的关键因素，在现实情况和有限条件下做好任务分解和进度安排<br>2）针对计划合理地调配和充分利用现有资源，解决项目中大部分问题<br>3）在活动过程中充分预见可能的问题，并提前确定相应的防范应变措施 |

## 二、专业知识

### 5. 设计类

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 理解并掌握基础的用研，体验设计，界面构建技能和方法 | 1）列举过往工作中主动实践到的用户研究、体验设计，界面构建方面的案例 |
| P3 | 理解并掌握主要的用研，体验设计，界面构建技能和方法 | 1）举证通过交互思维提升设计质量的案例及代表性的设计方案 |
| P4 | 全面了解各设计领域知识，具备动手能力 | 1）举证通过交互思维提升设计质量的案例及代表性的设计方案 |

### 6. 相关知识

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 了解产品、运营和开发的工作内容 | 1）熟悉所服务行业，并能做简单分析<br>2）熟悉产品策划、开发和运营的职能和基础知识，并能参与讨论<br>3）了解项目管理的基础概念，熟悉敏捷开发的流程，并能在项目中配合应用<br>4）对参与设计的产品、运营和开发工作内容的呈述符合 |
| P3 | 熟悉产品研发，运营全过程，有基本数据分析能力 | 1）对行业发展趋势有认识<br>2）对产品运营数据敏感，并能通过对数据分析来挖掘体验中的问题<br>3）了解设计方案的开发和运维成本<br>4）举证通过运营数据分析挖掘的体验问题<br>5）例举代表性的设计方案需要的开发成本、运维成本<br>6）对项目管理和敏捷开发的理解和呈述 |
| P4 | 能有效利用产品，开发，运营知识，推动项目成功 | 1）把握产业的发展趋势，并能做简单分析<br>2）对产品运营数据、市场表现敏感，并能通过分析来优化设计方案<br>3）能协调和推动产品、开发和运营来提升产品体验<br>4）掌握项目管理的能力，并能推动和改善项目的执行<br>5）举证通过运营数据分析梳理出的用户行为及挖掘出的用户需求<br>6）举证所制定的项目计划及执行情况的记录证明，并例举过程中解决的主要问题<br>7）例举通过主动协调多方角色带来产品体验提升的案例 |

## 三、专业技能

### 7. 设计定位

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 指导下把握需求，配合主设计师完成设计任务 | 1）能够从角色、场景、任务3个维度理解产品需求，并配合主设计师完成设计任务<br>2）熟悉同类产品，并能对其设计特点进行分析<br>3）介绍对一个重要需求的理解<br>4）举证同类产品的竞品分析 |
| P3 | 理解产品的整体需求，能够归纳和分解并进行针对的设计 | 1）能够负责完成小型项目的设计任务<br>2）通过分析需求独立完成对角色、场景、任务的定义<br>3）参与需求的制定过程，并能提出合理的优化建议<br>4）参与讨论和确定产品的设计理念，并在项目中积极执行<br>5）举证通过分析需求对角色、场景和任务所做出定义的案例<br>6）例举所参与制定的需求和产品的设计理念 |
| P4 | 准确把握需求，并主导确定设计目标，信息架构 | 1）能够独立负责完成中型项目的设计任务<br>2）能够甄别产品需求的合理性和准确性<br>3）对需求的制定过程有充分的话语权<br>4）主导确定产品的设计理念，并推动在项目中的具体实施<br>5）主导确定产品的信息架构和交互框架<br>6）例举所碰到的不合理的需求<br>7）举证所主导产品有关设计理念、架构性设计的提案 |

### 8. 设计实施

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 掌握常用工具，按时保质完成设计任务 | 1）掌握常用设计工具的基础用法：（Axure）<br>2）掌握基础的界面排版原则和方法<br>3）交互方案逻辑合理，流程清晰流畅<br>4）输出符合规范的交互设计稿，在评审中没有初级的可用性问题<br>5）举证具有代表性的设计方案，重点说明工具的使用情况 |
| P3 | 高效使用工具、方法、流程等，具备多样化设计技能和方法 | 1）能以Demo的形式对设计方案中的关键点进行演示<br>2）设计方案中能针对需求进行合理创新<br>3）熟悉掌握常用设计工具、方法<br>4）界面排版具备完整的逻辑和高质量的流程展示<br>5）举证代表性的设计方案，呈述其中的创新点，并对其中关键点以Demo形式作以演示 |
| P4 | 熟练并钻研工具、方法、流程等，有效提升效率，能提出行之有效的设计方案 | 1）以完整提案的形式输出产品的设计理念和框架性设计<br>2）设计方案能针对产品特点进行合理创新，在兼顾考虑产品需求、开发成本的前提下，驱动产品的体验提升<br>3）精通常用的设计工具、方法，并钻研设计流程帮助团队提高工作效率<br>4）例举通过钻研工具给团队带来的工作效率提升<br>5）举证代表性的设计方案和兼顾需求和开发成本的前提下所带来的体验提升<br>6）举证完整的设计提案<br>7）总监、项目组负责人就协助设计评审方面的评价 |

### 9. 设计总结

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 团队内设计总结分享 | 1）能够在团队内部进行设计心得，设计思路，设计方法等维度的分享与交流<br>2）在项目里程碑的阶段，对过程中积累的控件、组件的用法，常见的设计方法进行提炼总结，有阶段性书面总结输出<br>3）举证项目的阶段性书面总结案例 |
| P3 | 团队内设计总结，专题研究，规范制定 | 1）在项目里程碑的阶段，对设计方法和流程、工作效率等方面的经验进行有价值的总结，并影响到团队<br>2）能进行专业领域的专题性研究<br>3）参与基本的产品设计规范的制定<br>4）参与到项目相关的控件、组件范式规范的撰写<br>5）举证项目的总结性文档，并提供分享证明<br>6）举证所撰写的设计规范文档，并呈述过程 |
| P4 | 跨项目输出设计总结，专题研究，能制定中型项目规范 | 1）对项目中的经验进行总结并输出总结，在产品团队、其他部门进行分享<br>2）能进行行业领域的专题性研究<br>3）制定中型项目设计规范，并在多个项目中推动执行<br>4）举证项目的总结性课程，并提供分享证明<br>5）举证发表的文章<br>6）举证所推动的跨项目团队设计规范的文档，并呈述过程 |

## 四、组织影响力

### 10. 方法论建设

| 职级 | 关键词 | 行为标准 |
|---|---|---|
| P2 | 总结归纳 | 1）能够定期进行个人工作总结，不断优化工作 |
| P3 | 提炼规律 | 1）能从工作中总结与提炼共性的规律，把岗位的工作心得或案例沉淀总结并输出成果，形成可复制的经验与模式，优化工作效率 |
| P4 | 方法论沉淀 | 1）通过标杆研究及内部实践，在本专业领域内沉淀出切实有效的方法论，并推广应用 |

### 11. 知识传承

| 职级 | 关键词 | 行为标准 |
|---|---|---|

| P3 | 跨团队经验与知识的分享 | 1）积极参加部门内或部门间工作相关的交流和研讨，并进行经验与知识的分享及学习 |
| P4 | 营造分享的组织氛围 | 1）主动引导团队成员一起进行知识分享，营造主动学习、分享和共同进步的团队氛围<br>2）能够主导开发课程并进行授课培训 |

|---|---|---|

| P3 | 随时辅导 | 1）能够辅导1-2个初级员工，进行随时辅导，帮助改进工作效率，提升能力发展 |

## 职级岗位评估分打分参考（0～5）

将评估对象在上述12个能力项目上的整体表现，对照其职级（P2/P3/P4）的行为标准，综合判断匹配程度，给出分数：

| 分数区间 | 判断标准                            |
| ---- | ------------------------------- |
| 5    | 全面达到甚至超出所属职级标准，多数能力项已展现出上一职级的特征 |
| 4    | 达到所属职级标准，仅个别能力项稍有欠缺             |
| 3    | 基本达到所属职级标准，但存在多个能力项的明显短板        |
| 2    | 明显低于所属职级标准，仅达到下一职级（如P3对照P2标准）水平 |
| 1    | 大幅低于所属职级标准，多数能力项未达标             |
