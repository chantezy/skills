# 模块：客户管理 (Customer)

## 定位
**客户资产沉淀层**，所有客户数据的统一管理中心。

## 子功能详解

### 1. 客户列表
- 全部客户：所有可见客户（受权限控制）
- 我的客户：当前用户跟进的客户
- 公海客户：未分配 / 跟进超时回收的客户
- 回收站：删除恢复

**关键能力**：筛选、搜索、批量操作、字段自定义

### 2. 客户详情（360° 视图）
- 基本信息：昵称、手机、地区、来源、添加员工、添加时间
- 联系记录：聊天历史（基于消息存档）
- 订单记录：游戏盒平台订单 / 充值记录
- 标签画像：所有标签的可视化展示

### 3. 客户分组
- 分组管理：手动 / 自动分组
- 分组规则：基于标签 / 行为 / 属性的规则

### 4. 客户标签
- 标签体系：分级标签管理（角色 / 兴趣 / 活跃度 / 消费力 / 行为）
- 自动标签：基于规则 / 行为触发的自动打标

### 5. 客户导入/导出
- 批量导入手机号 / 客户信息
- 数据导出（受权限控制）

## 与其他模块的关系

```
企业微信模块 → 同步客户基础数据 → 客户管理
游戏盒APP → 同步行为/订单数据 → 客户管理
客户管理 → 提供客户+标签 → 客户运营
客户管理 → 提供原始数据 → 数据分析
```

## 适合发散的扩展方向

### 玩家专属
- 游戏兴趣画像（基于游戏盒行为）
- 玩家等级 / 价值分层（白嫖/小R/大R）
- 主玩游戏识别

### 创作者专属
- 创作者档案子视图（粉丝数/月发文量/收益）
- 签约状态管理
- 内容矩阵关联

### 公会专属
- 公会档案
- 成员管理
- 公会活跃度

### 通用增强
- 客户合并（多触点同一人识别）
- 客户黑名单
- 客户关系图谱（公会成员关系 / 创作者粉丝关系）

## 给 AI 的提示
- 这里管的是"是谁"，不是"做什么"
- 任何"管理客户、查看客户、给客户打标"都落在这
- 标签体系是基础设施，必须先规划再扩功能
