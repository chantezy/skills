# 4399游戏盒 SCRM 产品档案

> ⚠️ 这是 Skill 中最高优先级的上下文。
> 任何方案讨论前，AI 必须先了解我们现有产品有什么，避免"重复造轮子"或"架构冲突"。

## 产品基本信息
- **产品名**：4399游戏盒 SCRM 系统
- **测试环境**：https://test-scrm.haohaowan.net
- **产品类型**：游戏社区平台专用的社交客户关系管理系统
- **核心定位**：基于企业微信，承接 4399 游戏盒平台用户的运营底座

## 产品架构一览（6 大模块）

```
SCRM 系统
├── 工作台 Work          → 日常入口（首页、待办、消息）
├── 客户管理 Customer    → 客户资产（列表、详情、分组、标签、导入导出）
├── 客户运营 Operation   → 运营动作（群发、活动、客户旅程、内容中心）
├── 数据分析 Analytics   → 数据驱动（客户/运营/销售/报表）
├── 企业微信 WeCom       → 企微对接（员工、客户联系、消息存档、应用配置）
└── 系统设置 Settings    → 后台管理（企业/权限/日志）
```

## 三层架构定位

| 层级 | 模块 | 角色 |
|---|---|---|
| 入口层 | 统一登录 / 工作台 / 消息中心 | 用户进入系统的第一触点 |
| 业务层 | 客户管理 / 客户运营 / 企业微信 | 核心业务能力 |
| 支撑层 | 数据分析 / 内容中心 / 系统设置 | 为业务提供基础设施 |

## 数据流向

```
采集层（企微 / 游戏盒APP / 网页端 / API）
    ↓
处理层（客户归集 / 标签计算 / 行为分析 / 数据清洗）
    ↓
存储层（客户 / 行为 / 运营 / 日志数据）
    ↓
应用层（客户管理 / 客户运营 / 数据分析 / 系统管理）
```

> 📌 关键差异化：我们除了企微数据，还能采集 **游戏盒 APP 行为数据**，这是大多数通用 SCRM 没有的优势。

|---|---|---|
