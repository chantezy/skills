# 能力缺口与扩展方向（重点）

> 这是 AI 给方案最重要的参考——告诉 AI 我们「现在有什么、缺什么、可以怎么补」。

## 四、差异化机会点（重点中的重点）⭐

### 1. 双数据源融合
我们有 **企微 + 游戏盒 APP** 双数据，通用 SCRM 拿不到 APP 行为。
**机会**：基于 APP 行为的实时触发（"用户启动 XX 游戏 30 分钟未充值 → 触发优惠券"）

### 2. 游戏行业垂类深耕
通用 SCRM 是横向的，我们是纵向的。
**机会**：礼包码、内测、CPS 归因、公会工具——这些是通用 SCRM 永远做不深的。

### 3. 创作者经济
我们既有玩家，又有创作者。
**机会**：把 SCRM 升级为"创作者运营平台"，撮合广告 / CPS 订单。

### 4. 公会组织节点
游戏行业独有的强组织节点。
**机会**：公会工具包小程序 + 公会数据看板 + 公会扶持计划。

## 六、给 AI 的方案设计指引

### 当用户提出需求时，AI 应：

1. **先在"已有能力"中找** → 能复用就复用
2. **再在"能力缺口"中找** → 标注优先级和落点
3. **最后挖掘"差异化机会"** → 利用双数据源 / 游戏垂类深耕

### 输出建议模板

```
方案 X：XX 功能
- 落点模块：客户运营 → 营销活动（新增子模块"礼包码发放"）
- 复用现有能力：内容中心的素材库、客户管理的标签筛选
- 新建能力：礼包码批量导入、码值池管理、发放规则引擎
- 数据流：开发商 API → 码值池 → 发放任务 → 客户领取 → 归因
- 涉及角色：运营经理（创建发放任务）、运营专员（执行）
- 工作量预估：前端 8d、后端 10d、数据 3d
```

## 七、维护机制
- 每季度盘点能力清单
- 每次新功能上线后更新"已有能力"
- 业务方提出新需求时同步更新"能力缺口"
