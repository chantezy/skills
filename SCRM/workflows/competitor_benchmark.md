# 竞品对标工作流

## Step 1: 确认对标对象
- 是对标整个竞品 vs 对标某个功能模块
- 用户当前的位置（已选定方案 / 还在调研）

## Step 2: 加载竞品资料
- 检索 `references/competitors/{竞品名}.md`
- 若涉及多家，逐家拆解

## Step 3: 拆解维度

| 维度 | 说明 |
|---|---|
| 产品定位 | 一句话讲清差异化 |
| 核心功能 | 列出 3–5 个亮点功能 |
| 技术实现 | 推测依赖的企微接口 + 自建模块 |
| 目标客户 | 行业 + 规模 |
| 商业模式 | 收费方式 / 客单价区间 |
| 优势 | 3 条 |
| 劣势 | 3 条 |

## Step 4: 差异化结论

- 我们应该「学」什么
- 我们应该「不做」什么
- 我们应该「做得不一样」的点

## Step 5: 落地建议
给出 3 条可立即采取的行动。
