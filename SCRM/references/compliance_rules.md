# SCRM 合规红线清单

## 一、企微平台规则红线

1. ❌ 严禁使用非官方 API / 模拟器多开 / 破解版（封号 + 法律风险）
2. ❌ 不得未经客户同意拉群、群发骚扰
3. ❌ 单员工高频加好友（>200/天）→ 风控
4. ❌ 朋友圈含诱导分享 / 集赞 / 关注 → 限流封号
5. ❌ 群内发送外链跳转外部支付 → 高风险
6. ✅ 必须通过官方接口实现所有能力

## 二、数据隐私合规（《个人信息保护法》）

1. 收集客户手机号 / 身份信息 → 必须有明示同意
2. 会话存档 → 员工授权 + 客户知情（页面显示）
3. 标签包含敏感信息（疾病、收入）→ 需特殊授权
4. 跨境数据传输 → 严格管控
5. 客户有权要求删除数据

## 三、行业特殊合规

### 金融
- 投顾话术须备案，禁止"收益承诺"
- 风险评测必须完成才能推产品
- 会话存档强制 + 双录（部分场景）

### 医疗 / 医美
- 不得发布治愈率 / 有效率宣传
- 价格信息须明确
- 处方药严禁线上销售引导

### 教育
- 不得承诺"提分""保过"
- K12 学科类有严格的私域投放限制

### 保险
- 销售人员必须持证
- 产品说明必须完整披露

## 四、运营风险红线

| 风险类型 | 表现 | 后果 |
|---|---|---|
| 飞单 | 员工引导客户加个人微信 | 客户流失 + 公司损失 |
| 敏感词 | 承诺 / 辱骂 / 竞品攻击 | 投诉 / 法律风险 |
| 数据泄露 | 客户信息外传 | 法律责任 |
| 过度营销 | 高频群发骚扰 | 投诉 + 封号 |

## 五、合规自查 Checklist（方案上线前必过）

- [ ] 接口全部使用官方
- [ ] 加好友 / 群发频次符合规则
- [ ] 收集个人信息有授权文案
- [ ] 会话存档已开员工授权
- [ ] 话术库经过法务 / 业务审核
- [ ] 敏感词库已配置
- [ ] 离职继承流程已建立
- [ ] 客户行使删除权的入口已开放

## 六、给 AI 的提示

- 用户方案涉及"加好友 / 群发 / 朋友圈 / 数据收集 / 会话存档"时，主动加载本文件。
- 输出时单独列"合规与风险"板块，标红高风险点。
- 金融 / 医疗 / 教育 / 保险 行业额外校验对应小节。
