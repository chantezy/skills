# SCRM 专业术语词典

> 使用说明：当 AI 在对话中遇到下列术语时，应自动加载该术语的功能要素和落地映射，避免空谈概念。

## 一、引流获客类

### 活码（Channel QR Code）
- **定义**：可动态变换承接人 / 群的二维码，扫码后按规则分流。
- **业务价值**：解决"二维码失效""单人加爆""无法统计渠道"。
- **典型场景**：包裹卡、门店海报、广告落地页、线下地推。
- **功能要素**：
  - 承接人池（轮流 / 权重 / 按地域 / 按标签）
  - 个性化欢迎语（支持变量：渠道名、客户昵称）
  - 自动打标（来源渠道标签）
  - 加满自动切换 / 失效切换
  - 渠道数据统计（扫码量 / 添加量 / 通过率）
- **关联接口**：`externalcontact/add_contact_way`
- **关联功能**：渠道追踪、自动打标、SOP 触发
- **常见坑**：单员工"联系我"数量上限；通过率低多半是头像 / 话术问题
- **竞品差异**：微伴支持时段分流；微盛支持 AI 个性化欢迎语

### 群活码
- **定义**：自动生成新群、群满自动切换下一个群的二维码。
- **功能要素**：群满 200 自动建新群、群欢迎语、群标签、自动入群打标
- **关联接口**：`externalcontact/groupchat/*`
- **坑**：企微一个客服最多 2000 群，需要多客服轮转

### 任务宝（Task Pro）
- **定义**：邀请 N 位好友扫码 → 解锁福利的裂变玩法。
- **功能要素**：裂变海报生成、邀请关系链追踪、奖品发放、防作弊
- **关键指标**：K 因子、裂变层级、ROI
- **坑**：奖品成本失控、羊毛党、企微对裂变行为有反作弊

### 渠道追踪与归因
- **定义**：从扫码 → 加好友 → 转化的全链路渠道来源识别。
- **功能要素**：UTM 参数、渠道标签、首次来源 vs 末次来源、多触点归因
- **指标**：CAC（获客成本）、渠道 ROI

## 二、客户管理类

### 客户档案（Customer Profile）
- **定义**：以客户为中心聚合多维度数据的统一视图。
- **功能要素**：
  - 基础信息（昵称、手机、性别、地区）
  - 标签体系（系统标签 + 业务标签）
  - 来源信息（渠道、添加员工、添加时间）
  - 互动轨迹（消息、朋友圈互动、群活跃）
  - 交易数据（订单、客单价、复购）
- **数据来源**：企微会话存档、业务系统、自动埋点
- **关联接口**：`externalcontact/get_external_contact`

### 客户标签
- **定义**：对客户进行多维度分类的"标记"，是 SCRM 一切运营的基础。
- **分类**：
  - 系统标签：性别、地区、来源等
  - 业务标签：意向度、价值等级、产品偏好
  - 行为标签：点击过 XX、浏览过 XX、参与过 XX
- **打标方式**：手动 / 关键词触发 / 行为触发 / AI 语义识别
- **关联接口**：`externalcontact/mark_tag`
- **坑**：标签体系混乱是 SCRM 头号问题，必须先规划标签 schema

### 客户分层 / RFM
- **定义**：基于 Recency（最近购买）/ Frequency（频次）/ Monetary（金额）三个维度对客户分层。
- **落地为功能**：
  1. 自动计算 RFM 三个分值
  2. 客户分层视图（8 类客户）
  3. 分层 SOP 自动触发
  4. 分层营销活动
- **场景**：零售、电商最常用

### 公海 / 私海池
- **定义**：未分配 / 跟进超时的客户进入公海；已认领的进入员工私海。
- **功能要素**：自动回收规则、抢客规则、跟进超时提醒
- **场景**：有销售团队的业务必备

### 客户生命周期（Customer Lifecycle）
- **定义**：从陌生 → 潜在 → 新客 → 活跃 → 沉睡 → 流失 的阶段划分。
- **功能要素**：阶段自动流转规则、阶段 SOP、阶段转化率统计

## 三、运营触达类

### SOP（Standard Operating Procedure）
- **定义**：标准化的运营动作时序方案。
- **类型**：
  - 客户 SOP：T+N 日给特定标签客户推消息
  - 群 SOP：群运营节奏（早安 / 活动 / 答疑）
  - 朋友圈 SOP：定时分组发圈
- **功能要素**：触发条件、时间序列、内容素材、发送对象、执行提醒
- **关联接口**：`externalcontact/add_msg_template`
- **坑**：模板化陷阱——所有客户一套 SOP，必须按标签分层

### 群发助手
- **定义**：批量给客户 / 群发送消息，需员工二次确认。
- **限制**：企微规定每个员工每天对每个客户只能群发 1 次
- **关联接口**：`externalcontact/add_msg_template`

### 朋友圈（企业朋友圈）
- **定义**：管理员统一编辑、员工一键发送的朋友圈。
- **功能要素**：分组可见、定时发布、数据统计（点赞 / 评论 / 点击）
- **关联接口**：`externalcontact/add_moment_task`
- **限制**：每个员工每天 1 条企业朋友圈

### 素材库
- **定义**：统一管理图文 / 视频 / 海报 / 小程序 / 文章的内容资产中心。
- **功能要素**：素材分类、版本管理、使用统计、点击追踪
- **价值**：所有 SOP / 群发 / 朋友圈的内容都从这里调用

### 侧边栏（企微导购侧边栏）
- **定义**：在企微聊天窗口右侧显示的应用，给一线员工提供工具。
- **功能要素**：
  - 客户档案速览
  - 话术推荐
  - 商品推荐
  - 优惠券发放
  - 订单查询
- **关联接口**：企微 JS-SDK + 自建应用

## 四、转化复购类

### 导购 / 企微导购
- **定义**：在企微上承担 1V1 销售职能的角色（线下导购 / 销售顾问）。
- **赋能工具**：侧边栏、话术库、商品库、优惠券、订单查询、智能推荐

### 商城打通
- **定义**：把电商 / 小程序商城的客户行为同步进 SCRM。
- **打通点**：
  - 浏览行为 → 自动打标
  - 加购未付 → 触发提醒 SOP
  - 下单 → 订单标签 + 复购 SOP
  - 退款 → 风险预警

### 流失预警
- **定义**：基于客户行为预测流失风险并提前干预。
- **算法**：基于 RFM + 互动频率 + 时间衰减
- **落地**：流失分数 → 高分客户进召回 SOP

### 复购召回
- **定义**：针对沉睡 / 流失客户的唤醒动作。
- **功能要素**：召回话术 SOP、专属优惠券、新品通知

## 五、AI 智能化类

### 智能话术推荐
- **定义**：基于客户档案 + 会话上下文，给员工实时推荐话术。
- **技术**：意图识别 + RAG + 大模型
- **关键**：必须有审核兜底，避免 AI 幻觉

### 智能质检
- **定义**：基于会话存档自动检测违规、机会、风险。
- **检测项**：飞单、辱骂、敏感词、客户投诉、商机识别
- **价值**：管理者从抽检 5% 升级为全量质检

### AI 客户洞察
- **定义**：从会话中自动提炼客户画像、意图、偏好。
- **输出**：自动打标 + 客户摘要 + 下一步建议

### AI Agent 工作台
- **定义**：员工通过自然语言指挥 AI 完成业务动作。
- **示例**：「帮我给所有意向客户发新品介绍」→ AI 调用群发工具

## 六、合规与管理类

### 会话存档
- **定义**：企微官方提供的合法记录员工 - 客户聊天的能力。
- **要求**：员工授权 + 客户知情（聊天窗显示"对话被记录"）
- **关联接口**：`msgaudit/*`
- **场景**：金融、医疗强制；其他行业用于质检和飞单管控

### 离职继承 / 在职继承
- **定义**：员工离职 / 调岗时把客户关系转移给其他员工。
- **关联接口**：`externalcontact/transfer_customer`
- **坑**：客户有 3 天反悔期；群主继承单独处理

### 飞单
- **定义**：员工把客户引到个人微信、私下交易，绕过公司。
- **检测**：会话存档 + 敏感词（"私聊""加我微信""线下打款"）
- **处置**：告警 → 调查 → 处罚

### 敏感词监控
- **定义**：实时扫描员工对客发言，命中敏感词触发告警。
- **常见词**：竞品名、价格承诺、夸大宣传、个人联系方式

## 七、数据指标类（与 metrics_dictionary.md 联动）

> 完整指标口径见 `metrics_dictionary.md`

| 指标 | 定义 | 公式 |
|---|---|---|
| 加粉成本（CAC） | 获取一个企微好友的成本 | 渠道花费 / 新增好友数 |
| 通过率 | 添加请求中被通过的比例 | 通过数 / 申请数 |
| 触达率 | 群发被实际看到的比例 | 已读数 / 发送数 |
| 转化率 | 私域客户中下单的比例 | 下单客户数 / 私域总客户数 |
| LTV | 单个客户全生命周期价值 | 平均客单价 × 复购次数 × 毛利率 |
| K 因子 | 裂变系数 | 平均每个用户带来的新用户数 |
