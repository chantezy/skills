# 用户角色画像

> 平台 SCRM 设计的起点，所有 SOP / 功能 / 活动都应基于角色分层。

## 一、普通玩家（C 端主体，占比 80%+）

### 细分子类
| 子类 | 特征 | 核心诉求 | 运营策略 |
|---|---|---|---|
| 路人玩家 | 偶尔来看攻略 / 找游戏 | 找内容、找游戏 | 内容推送 + 个性化推荐 |
| 兴趣玩家 | 关注 1-3 款游戏 | 跟进游戏动态、找同好 | 游戏垂类群 + 福利 |
| 硬核玩家 | 重度投入、深度内容消费 | 高质量讨论、专业内容 | 核心玩家群 + 内测特权 |
| 氪金玩家 | 高消费意愿 | 礼包、首发、稀有道具 | 商城 + 1V1 福利推送 |

### 私域典型场景
- 攻略求助 → 自动回复 / 群答疑
- 新游推荐 → 个性化兴趣群
- 礼包领取 → 商城跳转
- 活动报名 → 小程序 / H5

## 二、内容创作者（C+B 双重身份）

### 细分子类
| 子类 | 特征 | 核心诉求 | 运营策略 |
|---|---|---|---|
| 萌新创作者 | 偶尔发内容、求曝光 | 教程、流量入门 | 新人创作者群 + 课程 |
| 腰部创作者 | 稳定产出、有小规模粉丝 | 流量扶持、变现入门 | 流量计划 + 商业化对接 |
| 头部 KOL | 大粉丝量、高产能 | 商业化、独家合作 | 1V1 经纪人 + 签约 |
| MCN | 多账号矩阵 | 批量化运营、平台合作 | B 端商务对接 |

### 私域典型场景
- 流量扶持申请
- 选题/热点同步
- 数据分析与复盘
- 商业化订单对接

## 三、公会 / 战队会长

### 特征
- 手握 50-500 名成员
- 自带流量与活跃度
- 强组织能力

### 核心诉求
- 公会管理工具（报名、签到、活动）
- 平台流量曝光
- 赛事/活动承接

### 运营策略
- 公会会长群（高频对接）
- 公会工具包（小程序）
- 优质公会扶持计划

## 四、游戏开发商（B 端）

### 细分子类
| 子类 | 特征 | 核心诉求 | 运营策略 |
|---|---|---|---|
| 中小独立游戏团队 | 缺曝光、缺测试用户 | 平台曝光、首发支持 | 入驻通道 + 商务 BD |
| 中大型厂商 | 有预算、有节奏 | 营销投放、活动联动 | 商务团队 + 服务方案 |
| 海外游戏 | 进中国市场需要本地化 | 国服首发、文化适配 | 海外 BD + 本地化方案 |

### 私域典型场景
- 商务对接
- 投放方案
- 活动联动
- 数据复盘

## 五、活跃用户标签矩阵（建议落地的标签 schema）

```
角色身份（一级）
├─ 玩家
│   ├─ 兴趣偏好（游戏类型：MOBA/FPS/二次元/SLG/独立游戏…）
│   ├─ 活跃度（潜水/普通/活跃/核心）
│   ├─ 消费力（白嫖/小额/中额/大R）
│   └─ 内容偏好（攻略/视频/直播/评测）
├─ 创作者
│   ├─ 层级（萌新/腰部/头部/MCN）
│   ├─ 内容类型
│   └─ 合作状态（自由/签约/独家）
├─ 公会会长
│   ├─ 公会规模
│   ├─ 主玩游戏
│   └─ 活跃度
└─ 开发者
    ├─ 公司规模
    ├─ 游戏类型
    └─ 合作阶段
```

## 给 AI 的提示
- 用户提到"用户运营"，先问"哪类用户"
- 不同角色用 **不同的企微账号矩阵** 服务（玩家客服号 / 创作者经纪人号 / 公会运营号 / 商务号）
- 标签体系是一切运营的基础，必须先建
